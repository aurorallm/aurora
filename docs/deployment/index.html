Standard Installation

REF: 01A-DEPL
01

Pull the Image

Use Docker for production-style local runs and container platforms. The image supports all configuration via environment variables.

02

Command Execution

Pull the official production image and link your gateway configuration file. Ensure your provider API keys are accessible as environment variables.

# Step 1: Run Container
docker run -d --name aurora -p 8080:8080 \
  -e AURORA_MASTER_KEY="your-secure-key" \
  -e GROQ_API_KEY="gsk_..." \
  aurorahq/aurora

The config.yaml

REF: CONFIG_STRUCT

Aurora uses a declarative YAML schema. Hot-reloads are triggered upon file mutation without dropping active inference connections.

# Global Server Settings
server:
  port: 8080
  mode: production
  metrics_enabled: true

# Define Model Providers
providers:
  - name: gpt-4-primary
    type: openai
    weight: 80
  - name: anthropic-fallback
    type: anthropic
    weight: 20

# Intelligent Routing Rules
routing:
  strategy: least_latency
  fallbacks: enabled

Operational Intelligence

Health Checks

Implement liveness and readiness probes pointing to /health. Aurora reports internal provider status alongside binary health.

Resource Guarding

Set memory limits strictly. Go-Native Aurora is highly efficient; 256MB is often sufficient for 1000+ concurrent streams.

PII Masking

Enable the native scrubbing middleware in production to prevent upstream data leaks.

Secret Injection

Never hardcode keys. Use Kubernetes Secrets or AWS Secrets Manager linked via environment injection.

Debug Protocol

REF: ERR_LOGS

Error: Provider Timeout

Verify egress network rules. Ensure outbound 443 access to API endpoints.

Error: Invalid Auth Payload

Check env var prefixing. Verify API keys are correctly scoped for models.
